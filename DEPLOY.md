# Deploying Song Roulette to spotifyroulette.com (Render)

The app is one always-on Node process: Express + Socket.io serving the built
React client. It keeps rooms and sessions **in memory** and uses WebSockets, so
it must run on a persistent host — **not** Vercel/Netlify (serverless) or a
static host. Render's "Web Service" is the right fit.

## One-time setup

### 1. Push the repo to GitHub
The code (incl. `render.yaml`) is on `main` at `DhruvaKanna07/SpotifyRoulette`.

### 2. Create the Render service
1. Render dashboard → **New → Blueprint** → connect the GitHub repo.
2. Render reads `render.yaml` and proposes a web service `spotify-roulette`.
3. When prompted, set the secret env var:
   - **`SPOTIFY_CLIENT_ID`** = your Spotify app's client id (from `.env` locally).
   - `SESSION_SECRET` is auto-generated by Render; `NODE_ENV`, `CLIENT_ORIGIN`,
     `REDIRECT_URI` come from `render.yaml`.
4. Click **Apply**. First deploy runs `npm install && npm run build`, then
   `npm run start:prod`. Health check: `/api/health`.

At this point the app is live at `https://spotify-roulette.onrender.com` (or
similar). Test it there before wiring the domain.

### 3. Attach the custom domain in Render
1. Service → **Settings → Custom Domains → Add** `spotifyroulette.com` **and**
   `www.spotifyroulette.com`.
2. Render shows you the DNS targets to use (an apex A record / ALIAS and a `www`
   CNAME). Note them for the next step.

### 4. Point the domain's DNS (Squarespace)
Your domain currently points at Squarespace's own servers. To serve the app you
must repoint it. In the Squarespace/Google Domains DNS panel:

**Remove** (these send the domain to Squarespace, not your app):
- the four `A @` records → `198.49.23.144`, `198.49.23.145`, `198.185.159.144`,
  `198.185.159.145`
- the `CNAME www → ext-sq.squarespace.com`
- the `HTTPS @` record (the SVCB/HTTPS record hinting Squarespace's IPs)

**Add** what Render gave you in step 3, typically:
- `A  @  <Render's apex IP>` (or an ALIAS/ANAME if the registrar supports it)
- `CNAME  www  <your-service>.onrender.com`

**Leave untouched** (unrelated to hosting — these are email/verification):
- `TXT @  v=spf1 -all`
- `TXT _dmarc …`, `TXT _domainkey …`
- `CNAME _domainconnect …` (Squarespace management; harmless to keep)

DNS can take from minutes up to a few hours. Render auto-provisions a Let's
Encrypt TLS cert once DNS resolves — HTTPS is automatic, nothing to configure.

### 5. Update Spotify (required, or login breaks)
In the [Spotify developer dashboard](https://developer.spotify.com/dashboard)
for this app:
1. **Edit Settings → Redirect URIs** → add exactly
   `https://spotifyroulette.com/callback` (keep the local
   `http://127.0.0.1:5173/callback` too for dev). Must match character-for-char.
2. **User Management** → the app is in dev mode (max **5 users**). Add each
   friend's Spotify account email. Without this they can't log in.
3. The app owner needs Spotify **Premium** (embeds/playback).

If you later outgrow 5 users, apply for Spotify "extended quota" mode.

## Deploying updates
Push to `main` → Render auto-builds and redeploys. That's it.

## Notes / gotchas
- **Free tier sleeps** after ~15 min idle and cold-starts on the next request
  (~30s). For a party you're about to play, either open it a minute early or use
  the **Starter** plan ($7/mo) to keep it warm.
- **State is in-memory** — a redeploy or the free-tier sleep **wipes all rooms
  and logins**. Everyone reconnects Spotify and rejoins. Fine for casual play;
  don't redeploy mid-game.
- If login redirects fail, 99% of the time it's a `REDIRECT_URI` mismatch
  between Render's env var and the Spotify dashboard — they must be identical.
